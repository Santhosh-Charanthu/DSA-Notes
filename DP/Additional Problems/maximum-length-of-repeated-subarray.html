Maximum Length of Repeated Subarray

🔗 Problem

Pattern

  • DP on Arrays
  • Prefix vs Prefix DP
  • Decision + Optimization DP
  • Contiguous Matching DP
  • Similar to LCS, but for Subarrays (Contiguous)
🔍Key Observation

The problem asks:

Find the maximum length of a common subarray present in both arrays.

The keyword is:

Subarray

A subarray must be contiguous.

Unlike a subsequence, we cannot skip elements. Once a mismatch occurs, the current common subarray immediately breaks.

This is the biggest difference from LCS.

🧠Intuition

Imagine you're extending a common subarray.

At every pair of indices (i, j), ask:

"What is the length of the longest common subarray ending exactly at nums1[i] and nums2[j]?"

Notice the words:

ending exactly at

This is a local state, not the best answer seen so far.

Case 1: Elements Match

nums1[i] == nums2[j]

The current common subarray can be extended.

Move diagonally.

Reason:

If the previous elements were also part of a common subarray, simply extend it by one.

Case 2: Elements Don't Match

nums1[i] != nums2[j]

The current common subarray ends immediately.

Return:

0

Reason:

A contiguous subarray cannot continue after a mismatch.

Unlike LCS, we are not allowed to skip elements.

State

dp(i, j)

Represents:

Length of the longest common subarray ending exactly at nums1[i] and nums2[j].

This is the most important idea of the problem.

Notice that this is not:

Maximum common subarray in the prefixes.

It only describes subarrays ending at the current indices.

Transition

Elements Match

dp(i, j) = 1 + dp(i-1, j-1)

Reason:

Extend the previous common subarray.

Elements Don't Match

dp(i, j) = 0

Reason:

The contiguous sequence is broken.

Base Cases

One array is exhausted

i < 0 || j < 0

Return:

0

Reason:

No common subarray can exist if one array has no elements.

        │
Elements Equal?

Interview Intuition

The biggest mistake is treating this like LCS.

In LCS:

We are allowed to skip elements.

So on mismatch:

max(up, left)

works.

In this problem:

We are not allowed to skip elements because the subarray must remain contiguous.

Therefore, once a mismatch occurs:

Current Common Subarray = Broken

So,

dp(i, j) = 0

instead of max(up, left).

Why We Need a Global Maximum

The DP state only tells us:

Length of the common subarray ending at (i, j)

The longest common subarray may end anywhere in the arrays.

Example:

nums1 = [1,2,3,2,1]
nums2 = [5,2,3,2,7]

The answer is:

[2,3,2]

which ends before the last indices.

So we maintain:

answer = max(answer, dp(i, j))

while computing every state.

Why We Don't Return max(up, left)

Your recursion visits:

(i-1, j)

(i, j-1)

These calls are not part of the DP transition.

They are made only to visit every state and update the global maximum.

The current state depends only on:

(i-1, j-1)

So the recurrence is:

Match
    ↓
1 + Diagonal

Mismatch
    ↓
0

The answer is stored separately as the maximum value among all DP states.

Difference from LCS

Longest Common Subsequence (LCS) Longest Common Subarray
Subsequence Subarray (Contiguous)
Skipping allowed Skipping not allowed
State = Best answer in prefixes State = Answer ending at (i, j)
Match → 1 + diagonal Match → 1 + diagonal
Mismatch → max(up, left) Mismatch → 0
Answer = dp[n][m] Answer = Maximum value in the DP table
