Longest Common Subsequence (LCS)

Pattern

  • DP on Strings → Prefix vs Prefix DP
  • Two strings are compared.
  • Question asks for the longest subsequence common to both strings.

State

java
findLcs(i, j)

Represents:

Length of the Longest Common Subsequence between s1[0...i] and s2[0...j].

Choice Diagram

At every state (i, j) there are two cases.

Case 1: Characters Match

text
s1[i] == s2[j]

Since both characters are equal, they must be included in one LCS.

So,

text
Take both characters.

Answer = 1 + LCS of remaining prefixes.

Transition:

java
1 + findLcs(i-1, j-1)

Case 2: Characters Do Not Match

text
s1[i] != s2[j]

Both characters cannot be part of the same subsequence simultaneously.

So we have only two choices:

  1. Ignore current character of String 1.
  2. Ignore current character of String 2.

Take the better result.

Transition:

java
max(
    findLcs(i-1, j),
    findLcs(i, j-1)
)

Why don't we skip both?

We never call

java
findLcs(i-1, j-1)

when characters are different.

Reason:

If skipping both gives the optimal answer, then skipping either one first will eventually explore that state.

Example:

text
find(i-1, j)
        ↓
find(i-1, j-1)

find(i, j-1)
        ↓
find(i-1, j-1)

Calling find(i-1, j-1) directly is redundant.

Base Case

java
if(i < 0 || j < 0)
    return 0;

Reason:

If either string becomes empty, no common subsequence is possible.

LCS = 0.

Memoization

java
if(dp[i][j] != -1)
    return dp[i][j];

Avoids recomputation of already solved subproblems.

Recurrence

If characters match:

text
dp[i][j] = 1 + dp[i-1][j-1]

Else:

text
dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Complexity

Time

text
O(N × M)

Each state is computed only once.

Space

text
O(N × M)

Memoization table.

Recursion stack:

text
O(N + M)

Recognition Pattern

Whenever the question contains:

  • Two strings
  • Common
  • Subsequence
  • Longest

Immediately think:

text
Prefix vs Prefix DP

State:

text
dp(i, j)

DP Thinking Process

Whenever solving LCS, ask yourself:

1. What does my state represent?

text
LCS between prefixes
s1[0...i]
s2[0...j]

2. What choices do I have?

  • Characters match → Take both.
  • Characters don't match → Skip one character.

3. What is the base case?

One string becomes empty.

4. What am I optimizing?

Maximum length.

Therefore, use:

text
max()

instead of + or min().

Intuition

Think of two pointers standing at the end of both strings.

  • If the characters are equal, include them in the answer and move both pointers.
  • If they are different, one of the characters cannot belong to the optimal subsequence.
  • Try ignoring one character at a time and choose the better result.

Template

text
State:
dp(i,j)

Meaning:
Answer for prefixes s1[0...i] and s2[0...j]

Base Case:
i < 0 || j < 0 → 0

Transition:
If equal:
    1 + diagonal

Else:
    max(up, left)

Complexity:
Time  : O(N*M)
Space : O(N*M)
Note: Your implementation includes an extra base case: ``java if(i == 0 && j == 0) ` It works correctly, but it is not necessary. The single base case `java if(i < 0 || j < 0) return 0; `` is sufficient because the recursion naturally handles all edge cases.

One suggestion for your notes: keep the state definition, choices, transition, and intuition together. During interviews, being able to explain those four sections clearly is often more valuable than remembering the exact code.
