Uncrossed Lines

🔗 Problem

Pattern

  • DP on Arrays
  • Prefix vs Prefix DP
  • Decision + Optimization (Maximum) DP
  • Same Pattern as Longest Common Subsequence (LCS)
🔍Key Observation

The problem asks:

Draw the maximum number of connecting lines between two arrays such that no two lines cross.

The important observation is:

  • Two equal numbers can be connected.
  • Once two numbers are connected, their relative order must be preserved.
  • Preserving order is exactly the definition of a Subsequence.

Therefore,

Maximum Uncrossed Lines = Longest Common Subsequence (LCS) of the two arrays.

🧠Intuition

Imagine both arrays placed one below the other.

Whenever two equal numbers are connected, every future connection must occur after those indices; otherwise, the lines would cross.

Therefore, we only need to preserve the relative order of matched elements.

This is exactly how LCS works.

Case 1: Elements Match

nums1[i-1] == nums2[j-1]

Connect these two elements.

Since they are now matched, move to the previous elements in both arrays.

Reason:

Connecting equal elements increases the answer by 1, and preserving order guarantees that future lines will not cross.

Case 2: Elements Don't Match

nums1[i-1] != nums2[j-1]

We cannot connect these two elements.

We have two choices:

  • Ignore the current element of nums1.
  • Ignore the current element of nums2.

Take whichever choice produces more uncrossed lines.

Reason:

Since the current elements cannot be connected, one of them must be skipped.

State

dp(i, j)

Represents:

Maximum number of uncrossed lines that can be drawn using the first i elements of nums1 and the first j elements of nums2.

(If using 0-based recursion like your solution, the meaning becomes: maximum lines using nums1[0...i] and nums2[0...j].)

Transition

Elements Match

1 + dp(i-1, j-1)

Reason:

Connect the two equal elements and continue with the remaining prefixes.

Elements Don't Match

Choose the better option:

max(
    dp(i-1, j),
    dp(i, j-1)
)

Reason:

Skip one element and keep the option that yields the maximum number of lines.

Base Cases

One array is exhausted

i < 0 || j < 0

Return 0

Reason:

If either array has no elements left, no more lines can be drawn.

🧾Algorithm
Compare current elements
        │
Elements Equal?
├── Yes → Connect them
│          + Move diagonally
│
└── No
      │
      ├── Skip element from nums1
      └── Skip element from nums2
             │
        Take maximum

Complexity

  • Time: O(N × M)
  • Space: O(N × M)

Space Optimization:

Can be optimized to O(M) using two 1D arrays, exactly like LCS.

Recognition Pattern

If the question asks:

  • Maximum uncrossed lines
  • Match equal elements while preserving order
  • Maximum ordered matching between two arrays

Immediately think:

      ↓
Prefix vs Prefix DP
      ↓
Same as LCS
      ↓
Match?
├── Yes → 1 + Diagonal
└── No → max(Up, Left)

Interview Intuition

Think of every line as matching one element from each array.

The condition "lines must not cross" means:

Once you match two elements, all future matches must appear after those indices in both arrays.

That is exactly the definition of a Common Subsequence.

So instead of thinking about drawing lines, think:

"Find the longest common subsequence between the two arrays."

Once you recognize this transformation, the recurrence becomes identical to LCS:

  • Match → Connect the elements (1 + diagonal)
  • Mismatch → Skip one element (max(up, left))
