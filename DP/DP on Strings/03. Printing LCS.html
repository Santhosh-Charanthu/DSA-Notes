Print Longest Common Subsequence (LCS) - Intuition

Key Idea

The DP table stores the length of the LCS for every pair of prefixes. To print the LCS, we backtrack from the last cell (dp[n][m]) to the first cell.

Why start from dp[n][m]?

  • dp[n][m] contains the length of the final LCS.
  • Start from the complete answer and trace back how it was formed.

Backtracking Rules

1. Characters Match

text
s1[i-1] == s2[j-1]
  • This character is definitely part of the LCS.
  • Add it to the answer.
  • Move diagonally (i--, j--) because both characters have been used.

2. Characters Don't Match

text
s1[i-1] != s2[j-1]

The current characters cannot both belong to the LCS.

Compare:

text
dp[i-1][j]
dp[i][j-1]
  • If dp[i-1][j] > dp[i][j-1], move up (i--).
  • Otherwise, move left (j--).

Move towards the cell with the larger value because it preserves the longer LCS.

Why reverse the answer?

During backtracking, characters are collected from the end of the LCS to the beginning.

Example:

plaintext
LCS = "ACE"

Collected:
E
C
A

So reverse the collected string at the end to obtain the correct LCS.

⚙️ Algorithm
  1. Build the LCS DP table.
  2. Start from (n, m).
  3. While i > 0 and j > 0:
  • If characters match:
  • Add character.
  • Move diagonally.
  • Else:
  • Move to the neighboring cell (up or left) having the larger DP value.
  1. Reverse the collected characters.
  2. The reversed string is the Longest Common Subsequence.

Time Complexity

  • Building DP Table: O(N × M)
  • Backtracking: O(N + M)
  • Total: O(N × M)

Memory Trick

plaintext
Match    → Take character + Diagonal ↖

Mismatch → Move towards Larger DP value (↑ or ←)

Collected in reverse → Reverse at the end
