Edit Distance

🔗 Problem

Pattern

  • DP on Strings
  • Prefix vs Prefix DP
  • Decision + Optimization (Minimum Operations) DP
🔍Key Observation

The problem asks:

What is the minimum number of operations required to convert one string into another?

Allowed operations:

  • Insert
  • Delete
  • Replace

Since we need the minimum number of operations, this is an Optimization DP, so the recurrence uses min().

🧠Intuition

We compare both strings from the end of their prefixes.

At every step, we ask:

"Do the current characters already match?"

Case 1: Characters Match

word1[i-1] == word2[j-1]

No operation is required.

Simply move to the previous characters in both strings.

Reason:

The current characters are already equal, so they contribute nothing to the edit distance.

Case 2: Characters Don't Match

word1[i-1] != word2[j-1]

We have three possible operations.

1. Insert

Insert word2[j-1] into word1.

After insertion:

  • The current character of word2 is matched.
  • word1 stays at the same position because the inserted character handled the match.

Move to:

dp(i, j-1)

Cost:

1 + dp(i, j-1)

2. Delete

Delete word1[i-1].

After deletion:

  • Remove the current character from word1.
  • Continue matching the remaining characters.

Move to:

dp(i-1, j)

Cost:

1 + dp(i-1, j)

3. Replace

Replace word1[i-1] with word2[j-1].

After replacement:

  • Both current characters become equal.
  • Move diagonally.

Move to:

dp(i-1, j-1)

Cost:

1 + dp(i-1, j-1)

Choose the operation with the minimum cost.

State

dp(i, j)

Represents:

Minimum operations required to convert the first i characters of word1 into the first j characters of word2.

Transition

Characters Match

dp(i, j) = dp(i-1, j-1)

Reason:

No operation is needed.

Characters Don't Match

Choose the minimum among:

Insert  → 1 + dp(i, j-1)

Delete  → 1 + dp(i-1, j)

Replace → 1 + dp(i-1, j-1)

Take the minimum of all three.

Base Cases

Source becomes empty

i == 0

Return:

j

Reason:

The only way to convert an empty string into a string of length j is to insert all j characters.

Target becomes empty

j == 0

Return:

i

Reason:

The only way to convert a string of length i into an empty string is to delete all i characters.

🧾Algorithm
Compare current characters
        │
Characters Match?
├── Yes → Move diagonally
└── No
      │
      ├── Insert
      ├── Delete
      └── Replace
            │
Take minimum cost
        │
Store in DP

Complexity

  • Time: O(N × M)
  • Space: O(N × M)

Space Optimization:

Since each state depends only on the previous row and the current row, it can be optimized to O(M) space.

Interview Intuition

Think of standing at the last character of both prefixes.

If the characters are already the same, no work is needed—just move to the previous characters.

If they are different, ask:

"What is the cheapest way to make these prefixes equal?"

You have exactly three choices:

  • Insert the required character.
  • Delete the extra character.
  • Replace the wrong character.

Each operation costs 1, so recursively compute the remaining cost and choose the minimum.

The key observation is:

  • Matching characters require no operation.
  • Mismatching characters force exactly one of the three edit operations.

That's why this problem is a Decision + Optimization DP that uses min() instead of max() or +.
