Minimum ASCII Delete Sum for Two Strings

🔗 Problem

Pattern

  • DP on Strings
  • Prefix vs Prefix DP
  • Decision + Optimization (Minimum Cost) DP
  • Edit Operations DP
🔍Key Observation

The problem asks:

Delete characters from both strings so that they become equal while minimizing the total ASCII value of deleted characters.

Unlike Delete Operation for Two Strings, where every deletion costs 1, here:

  • Every character has a different deletion cost (its ASCII value).

So this becomes a minimum cost deletion problem.

🧠Intuition

Imagine comparing the two strings from the end of their prefixes.

At every step, ask:

"What is the minimum ASCII deletion cost to make these prefixes equal?"

There are two possibilities:

  • The current characters are equal.
  • The current characters are different.

Case 1: Characters Match

str1[i-1] == str2[j-1]

Keep both characters.

Move diagonally.

Reason:

Since both characters are already equal, deleting either of them would only increase the cost unnecessarily.

Case 2: Characters Don't Match

str1[i-1] != str2[j-1]

One of the characters must be deleted.

Two choices:

Delete from String 1

ASCII(str1[i-1]) + dp(i-1, j)

Delete from String 2

ASCII(str2[j-1]) + dp(i, j-1)

Take the minimum cost.

Reason:

Eventually both strings must become equal, so at least one of the mismatching characters must be removed.

State

dp(i, j)

Represents:

Minimum ASCII deletion cost required to make the first i characters of str1 and the first j characters of str2 equal.

Transition

Characters Match

dp(i, j) = dp(i-1, j-1)

Reason:

No deletion is needed.

Characters Don't Match

dp(i, j) = min(
    ASCII(str1[i-1]) + dp(i-1, j),
    ASCII(str2[j-1]) + dp(i, j-1)
)

Reason:

Delete one character and choose the cheaper option.

Base Cases

First string is empty

i == 0

Delete every remaining character from str2.

ASCII(str2[0]) + ASCII(str2[1]) + ...

Reason:

The only way to make an empty string equal to a non-empty string is to delete every remaining character.

Second string is empty

j == 0

Delete every remaining character from str1.

ASCII(str1[0]) + ASCII(str1[1]) + ...

Reason:

All remaining characters must be removed.

Interview Intuition

This problem is almost the same as Delete Operation for Two Strings, but the cost model is different.

Delete Operation for Two Strings

Every deletion costs:

1

So we count the number of deletions.

Minimum ASCII Delete Sum

Every deletion costs:

ASCII(character)

So instead of minimizing the number of deletions, we minimize the total deletion cost.

The recurrence remains almost identical:

  • Match → Keep both characters.
  • Mismatch → Delete from either string and choose the cheaper option.

Core Difference

The structure of the DP is identical. The only change is the cost function:

  • Uniform cost → add 1
  • Weighted cost → add ASCII(character)

Whenever you see different costs associated with operations, think of it as a weighted version of the original DP problem.
