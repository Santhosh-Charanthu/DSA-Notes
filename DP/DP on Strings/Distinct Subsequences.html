Distinct Subsequences

Pattern

  • DP on Strings
  • Prefix vs Prefix DP
  • Decision + Counting DP

Key Observation

The problem asks:

How many distinct ways can we form string t as a subsequence of string s?

Unlike LCS, where we maximize the length, here we count all possible valid ways.

The biggest intuition change is:

  • Optimization Problem → Use max() / min()
  • Counting Problem → Use +

Intuition

We process both strings from left to right (using prefixes).

At every character of s, we decide whether to use it to match the current character of t or ignore it.

Case 1: Characters Match

s[i-1] == t[j-1]

We have two valid choices:

  1. Take the character and match it with t[j-1].
  2. Skip the character and look for another occurrence later in s.

Why both?

Because there may be multiple occurrences of the same character in s, and each choice can produce a different valid subsequence.

Example:

s = "aaa"
t = "a"

We can form "a" by choosing:

  • First 'a'
  • Second 'a'
  • Third 'a'

Total = 3 ways

Therefore,

Answer = Take + Skip

Case 2: Characters Don't Match

s[i-1] != t[j-1]

We cannot match these characters.

The only option is to skip the current character of s and continue searching.

State

dp(i, j)

Represents:

Number of ways to form the first j characters of t using the first i characters of s.

Transition

Characters Match

Take current character
+
Skip current character

Meaning:

dp(i, j) =
dp(i-1, j-1)
+
dp(i-1, j)

Characters Don't Match

Only one choice:

Skip current character

Meaning:

dp(i, j) =
dp(i-1, j)

Base Cases

Target becomes empty

j == 0

Return 1

Reason:

An empty string can always be formed by choosing nothing.

Source becomes empty

i == 0

Return 0

Reason:

A non-empty target cannot be formed from an empty source.

⚙️Algorithm
Start from both string lengths
        │
Characters Match?
├── Yes → Take + Skip
└── No  → Skip only
        │
Store result in DP
        │
Return total number of ways

Complexity

  • Time: O(N × M)
  • Space: O(N × M)

Space Optimization:

Since each row depends only on the previous row, it can be optimized to O(M).

Recognition Pattern

If the question asks:

  • Count the number of subsequences
  • Number of ways to form one string from another
  • Count possible transformations

Immediately think:

DP on Strings
      ↓
Prefix vs Prefix DP
      ↓
Decision + Counting
      ↓
Match?
├── Yes → Take + Skip
└── No  → Skip
      ↓
Count all ways using +

Interview Intuition

Think of every character in s as asking:

"Should I be used to match the current character of t, or should I be ignored so that a future occurrence might be used instead?"

Whenever characters match, both decisions are valid, so we count both.

Whenever they don't match, there is no decision—we simply skip the current character.

This is why the recurrence uses addition (+) instead of maximum (max), making it a Decision + Counting DP problem.
