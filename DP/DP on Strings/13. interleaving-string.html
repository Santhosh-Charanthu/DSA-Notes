Interleaving String

🔗 Problem

Pattern

  • DP on Strings
  • Prefix vs Prefix DP
  • Decision + Boolean DP
🔍Key Observation

The problem asks:

Can s3 be formed by interleaving s1 and s2 while preserving the relative order of characters in both strings?

Important points:

  • Every character of s3 must come from either s1 or s2.
  • The order of characters inside each string cannot change.
  • We only need to answer Yes/No, so this is a Boolean DP.
🧠Intuition

Imagine you are building s3 from left to right.

At any moment, you have already consumed:

  • some characters from s1
  • some characters from s2

Now ask:

Can I continue building the remaining part of s3?

At every step you have at most two choices:

  • Take the next character from s1.
  • Take the next character from s2.

If either choice successfully forms s3, then the answer is True.

State (Most Important)

dp(i, j)

Represents:

Can the first i characters of s1 and the first j characters of s2 interleave to form the first i + j characters of s3?

This is the most important idea of the problem.

Notice what the state does not include.

There is no third index for s3.

Why don't we need a third index?

Suppose

i = 3
j = 5

How many total characters have already been used?

3 + 5 = 8

Therefore,

the next character we must match is always

s3[i + j]

So,

k = i + j

is completely determined by i and j.

It is not an independent state variable.

Therefore,

Instead of

dp(i, j, k)

we simply use

dp(i, j)

which saves memory and simplifies the DP.

State Visualization

Suppose

s1 = "ab"
s2 = "cd"
s3 = "acbd"

If

i = 1
j = 2

then we have already consumed

"a" from s1

"cd" from s2

Total consumed characters:

i + j = 3

So we have already formed

"acd"

The next character to match is

s3[3]

Everything is determined using only (i, j).

Transition

Let

k = i + j

be the current position in s3.

Choice 1

Take the next character from s1.

Possible only if

s1[i] == s3[k]

Then move to

(i+1, j)

Choice 2

Take the next character from s2.

Possible only if

s2[j] == s3[k]

Then move to

(i, j+1)

Final Transition

takeFromS1 OR takeFromS2

Reason:

If either path successfully forms s3, then the answer is True.

Base Cases

Length Check

Before recursion,

if(s1.length() + s2.length() != s3.length())

Return

false

Reason:

The total number of characters must match.

Both Strings Exhausted

i == n

&&

j == m

Return

true

Reason:

All characters from both strings have been successfully used to build s3.

        │
      k = i + j

Complexity

Memoization

  • Time: O(N × M)
  • Space: O(N × M)

Tabulation

  • Time: O(N × M)
  • Space: O(N × M)

Space Optimized

  • Time: O(N × M)
  • Space: O(M)

Recognition Pattern

If the problem contains:

  • Two source strings
  • One target string
  • Preserve relative order
  • Can we form the target?

Immediately think:

Interview Intuition

Think of constructing s3 character by character.

At every step,

the next character of s3 can come from only one of two places:

  • s1
  • s2

If the current character of s1 matches s3, we may use it.

If the current character of s2 matches s3, we may use it.

If either choice succeeds, then the interleaving is possible.

That is why the recurrence uses:

OR

instead of

max

min

+

Why OR?

This is a Decision DP.

The state asks:

Can we form the target?

So,

  • If taking from s1 works → True
  • If taking from s2 works → True

Therefore,

answer = takeFromS1 || takeFromS2
