1092. Shortest Common Supersequence (SCS)🔗 Problem Link

Key Observation

A Shortest Common Supersequence is the shortest string that contains both strings as subsequences.

The Longest Common Subsequence (LCS) represents the characters common to both strings.

  • Common characters should be added only once.
  • Non-common characters must also be included.

Therefore,

text
Length of SCS = n + m - LCS

Intuition

Build the LCS DP table first, then backtrack to construct the answer.

1. Characters Match

text
str1[i-1] == str2[j-1]
  • This character is common to both strings.
  • Add it only once.
  • Move diagonally (i--, j--).

Reason: One occurrence is enough for both strings to remain subsequences.

2. Characters Don't Match

Compare:

text
dp[i-1][j]
dp[i][j-1]
  • If moving up, append str1[i-1] and move up.
  • If moving left, append str2[j-1] and move left.

Reason: The character being left behind is not part of the common subsequence, so it must be included in the supersequence.

3. Remaining Characters

If one string is exhausted, append all remaining characters of the other string.

Reason: These characters are unique and must appear in the final supersequence.

4. Reverse the Answer

Backtracking starts from (n, m) and collects characters from the end.

Reverse the collected string to obtain the correct SCS.

⚙️ Algorithm
text
Build LCS DP Table
        │
Start from (n, m)
        │
Match?
├── Yes → Add character once + Move Diagonal
└── No  → Add leaving character + Move to larger DP value
        │
Append remaining characters
        │
Reverse answer

Complexity

  • LCS DP Table: O(N × M)
  • Backtracking: O(N + M)
  • Total Time: O(N × M)
  • Space: O(N × M)

Recognition Pattern

If the question asks:

  • Shortest Common Supersequence
  • Merge two strings while preserving both as subsequences

Immediately think:

text
Build LCS DP
      ↓
Backtrack
      ↓
