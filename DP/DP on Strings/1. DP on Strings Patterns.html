📘 DP on Strings - Complete Pattern Notes (Interview + Competitive Programming)

Golden Rule Before solving any DP on Strings problem, answer these 5 questions: 1. How many strings are involved? 2. What does my DP state represent? 3. What choices do I have at each state? 4. What should I return? (Max / Min / Count / Boolean) 5. What is the smallest subproblem (Base Case)?

DP State

There are two equally valid definitions.

Version 1 (Prefix)

plaintext

=
Answer for

s1[0...i]

s2[0...j]

Recursive direction

plaintext
(i-1, j)

(i, j-1)

(i-1, j-1)

Version 2 (Suffix)

plaintext
dp(i, j)
=
Answer for

s1[i...end]

s2[j...end]

Recursive direction

plaintext
(i+1, j)

(i, j+1)

(i+1, j+1)

Both are correct.

Never mix the two definitions.

🔥 Top 1% DP Mindset

Never start coding immediately.

Spend 2–3 minutes answering:

✅ What is my state?

✅ What decision am I making?

✅ What are all the choices?

✅ Which choice gives the answer?

✅ What should I return?

If you can derive these five answers, you can solve almost every DP-on-Strings problem without memorizing recurrences.

Master Formula

Almost every DP-on-Strings problem can be solved using this thinking process:

plaintext
1. Identify the state.

↓

2. Stand at the current state.

↓

3. List every possible choice.

↓

4. Solve each smaller subproblem.

↓

5. Combine the answers using

max()

min()

+

||

&&

Memorize the thinking process, not the recurrence. That's what separates someone who solves only familiar problems from someone who can derive solutions in interviews and contests.
