Wildcard Matching

🔗 Problem

Pattern

  • DP on Strings
  • Prefix vs Prefix DP
  • Decision + Boolean DP (True/False)
🔍Key Observation

The problem asks:

Does the entire input string match the given wildcard pattern?

Wildcard symbols:

  • ? → Matches exactly one character.
  • * → Matches zero or more characters.

Since the answer is only True or False, this is a Boolean DP problem.

🧠Intuition

We compare both strings from the end of their prefixes.

At every step, we ask:

"Can the current characters be matched?"

Case 1: Characters Match

s[i-1] == p[j-1]

The current characters already match.

Move diagonally.

Reason:

Both characters are successfully matched.

Case 2: Pattern has '?'

p[j-1] == '?'

? represents exactly one character.

Consume one character from both strings.

Move diagonally.

Reason:

The current character of the string is matched by ?.

Case 3: Pattern has '*'

p[j-1] == '*'

* can behave in two different ways.

Option 1: '*' matches zero characters

Ignore the *.

Move left in the pattern.

dp(i, j-1)

Reason:

Treat * as an empty string.

Option 2: '*' matches one or more characters

Consume one character from the string but keep the *.

Move up.

dp(i-1, j)

Reason:

Since * can match multiple characters, it must remain available to match the next character as well.

If either option succeeds, the strings match.

Case 4: Characters Don't Match

s[i-1] != p[j-1]

and the pattern character is neither ? nor *.

Return False.

Reason:

No wildcard can help match these characters.

State

dp(i, j)

Represents:

Whether the first i characters of the string match the first j characters of the pattern.

Transition

Characters Match or '?'

dp(i, j) = dp(i-1, j-1)

'*'

Two choices:

Ignore '*'
OR
Use '*' to match current character

Meaning:

dp(i, j) =
dp(i, j-1)
OR
dp(i-1, j)

Characters Don't Match

dp(i, j) = false

Base Cases

Both string and pattern are exhausted

i == 0 && j == 0

Return True

Reason:

Two empty strings always match.

Pattern exhausted but string remains

j == 0 && i > 0

Return False

Reason:

There are still unmatched characters in the string.

String exhausted but pattern remains

i == 0 && j > 0

Return True only if every remaining character in the pattern is *.

Otherwise return False.

Reason:

Only * can represent an empty sequence.

🧾Algorithm
Compare current characters
        │
Match or '?'
├── Yes → Move diagonally
│
└── '*'
      │
      ├── Ignore '*'
      └── Match current character
             │
          Either works?
             │
           Return True
      │
Otherwise → False

Complexity

  • Time: O(N × M)
  • Space: O(N × M)

Space Optimization:

Possible to optimize to O(M) since only the previous row is required.

Recognition Pattern

If the question asks:

  • Wildcard matching
  • Pattern contains ? and *
  • Check if an entire string matches a pattern

Immediately think:

DP on Strings
      ↓
Prefix vs Prefix DP
      ↓
Decision + Boolean DP
      ↓
Match?
├── Character / '?' → Move diagonally
├── '*' → Zero OR One/More characters
└── Otherwise → False

Interview Intuition

Think of each wildcard as asking:

Normal Character

"I can match only the same character."

'?'

"I can match exactly one character."

'*'

"I have two choices."

  • Match nothing and disappear.
  • Match the current character and stay to potentially match more characters.

This is why the recurrence is:

dp(i, j-1)   // '*' matches zero characters

OR

dp(i-1, j)   // '*' matches one or more characters

Notice that * remains in the pattern when matching one or more characters because it may continue matching additional characters.

Difference from Regular Expression Matching

Do not confuse this problem with Regular Expression Matching.

Wildcard Matching

?  → Exactly one character
*  → Any sequence of characters

* is independent and can match any string.

Regular Expression Matching

.  → Exactly one character
a* → Zero or more occurrences of 'a'

Here, * always belongs to the previous character.

This difference changes the recurrence for the * case completely.
