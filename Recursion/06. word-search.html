Word Search

🔗 Problem

Pattern

  • Backtracking
  • DFS on Grid
  • Constraint-based Search
🧠Intuition

We need to determine whether the given word can be formed by moving up, down, left, or right in the grid without revisiting the same cell.

Start DFS from every cell matching the first character of the word. At each step, move only to a neighboring cell containing the next character. Mark the current cell as visited to avoid revisiting it, and backtrack after exploring all possible paths.

A small optimization is to check the character frequencies first. If the board doesn't contain enough occurrences of any character in the word, we can immediately return false without running DFS.

How to Think About Solving It

  1. Count the frequency of characters in the board and verify the board contains all characters required by the word.
  2. Traverse the board and start DFS from every cell matching the first character.
  3. Mark the current cell as visited.
  4. Explore all four directions to find the next character.
  5. If the entire word is matched, return true.
  6. Backtrack by unmarking the current cell before returning.
  7. If no path forms the word, return false.
⏱️Time Complexity
  • Time: O(N × M × 4^L)
    • N × M starting positions.
    • In the worst case, DFS explores up to 4^L paths, where L is the length of the word.
💾Space Complexity
  • Space: O(L + N × M)
    • O(L) recursion stack.
    • O(N × M) for the visited array.
